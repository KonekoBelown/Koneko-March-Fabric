# Koneko March v1.0.11

Koneko March 是一个面向 Minecraft Java Fabric 的战斗辅助 Mod。

本 Mod 围绕「行军」和「玉佩」两个核心附魔展开：玩家可以通过行军附魔召唤从军军团协助战斗，也可以通过玉佩附魔强化不死图腾的续命能力。它适合养老服务器的玩家和不想被原版所征服的玩家使用。

## 核心玩法

### 行军附魔

「行军」是一个武器附魔。

当玩家使用带有「行军」附魔的武器攻击目标时，会召唤一支从军军团协助玩家作战。从军会跟随玩家、攻击目标，并根据玩家的武器和装备形成不同的战斗风格。

正常生存环境下，「行军」最高为 5 级。每级会召唤 2 名从军。

如果通过命令或数据包获得更高等级的行军附魔，也可以生效。为了避免服务器过载，军团数量存在上限，最多保留 450 名从军，也就是225级。

## 从军行为

从军会继承玩家的装备和主副手武器。

从军可以：

- 使用近战武器攻击敌人。
- 跟随玩家移动。
- 在空旷区域组成较整齐的行军队形。
- 在狭窄道路、桥面或复杂地形中改为安全跟随。
- 在玩家骑马时生成骑兵单位。
- 在玩家下马后同步下马并清理军马。

行军队形会根据地形动态调整。空旷区域下，从军会更像阅兵式队列；在狭窄或容易卡住的地方，从军会优先保证可用性，而不是强行维持阵型。

## 骑兵系统

当玩家骑马时，行军召唤出的从军会以骑兵形式出现。

骑兵会拥有更适合骑乘单位的移动方式和跟随间距。在桥面、山路、洞穴等狭窄地形中，骑兵会尽量减少拥挤和卡位。

玩家下马后，骑兵也会同步下马，相关军马会被清理，避免留下无用实体。

## 玉佩附魔

「玉佩」是不死图腾附魔。

玩家可以通过铁砧将玉佩附魔添加到不死图腾上。带有玉佩的图腾会获得额外使用次数，次数等于3的等级次幂，暂没测试是否兼容五级以上的次幂。

## 主动护驾目标配置

如果安装了 Mod Menu，玩家可以在配置界面中管理从军的主动护驾目标。

配置界面会列出可主动攻击的实体 ID，玩家可以直接点击实体 ID 来启用或停用主动护驾。

配置界面支持：

- 查看可用实体 ID。
- 搜索实体 ID。
- 点击启用或停用目标。
- 恢复默认目标列表。
- 清空当前目标列表。
- 启用当前筛选结果中的实体。

多人服务器中，客户端配置会同步给服务器。服务器会根据每个玩家自己的配置决定该玩家从军的主动护驾行为。

## Koneko Warden Guard 联动

Koneko March 可以与 [Koneko Warden Guard](https://github.com/KonekoBelown/Koneko-Warden-Guard-Fabric/) 联动。

当同时安装 Koneko Warden Guard 时：

- 从军不会主动攻击监守者替身。
- 从军不会误伤带有 Koneko 友方保护标签的实体。
- 监守者替身也会把从军和军马视为友方单位。
- 两个 Mod 的召唤单位可以在同一战斗中共存。

Koneko Warden Guard 不是必需依赖。未安装时，Koneko March 仍可独立运行。

## 指令

可用指令：

- `/konekomarch recall`：收回当前军团。
- `/konekomarch release`：释放主手行军武器对应的军团。
- `/konekomarch cleanup`：管理员清理已追踪的行军实体。

## 依赖

必需：

- Minecraft `1.21.11`
- Fabric Loader `0.18.1+`
- Fabric API
- Java `21+`

可选：

- Mod Menu `17.0.0+`

## 安装方式

1. 安装 Fabric Loader。
2. 安装 Fabric API。
3. 将本 Mod 的 jar 文件放入 `.minecraft/mods` 文件夹。
4. 如需配置界面，可同时安装 Mod Menu。
5. 启动游戏。

## 注意事项

- 高等级行军附魔会召唤大量从军。虽然 Mod 已经加入数量上限和队形优化，但在大型整合包或低性能服务器上仍建议谨慎使用。
- 主动护驾目标列表会影响从军自动攻击哪些实体。如果出现误伤或目标选择不符合预期，可以在 Mod Menu 中调整。
- 从军会继承玩家装备，因此玩家的武器和装备会明显影响军团战斗表现。
- 本 Mod 可以独立运行；与 Koneko Warden Guard 同时安装时会启用友方识别联动。
